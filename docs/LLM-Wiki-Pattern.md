# 🧠 LLM Wiki Pattern: Концепция накопления знаний

## 1. Введение
LLM Wiki Pattern — это архитектурный подход к созданию долгоживущих баз знаний, где ИИ выступает не просто как поисковик, а как активный редактор и архитектор структуры.

### 1.2 Сравнение подходов

| Характеристика | Традиционный RAG | LLM Wiki Pattern |
| :--- | :--- | :--- |
| **Жизненный цикл** | Эфемерный (забывается) | Персистентный (сохраняется) |
| **Роль ИИ** | Поисковик фрагментов | Редактор и Архитектор базы |
| **Связи** | Векторный поиск (Top-K) | Явные Wiki-ссылки + Граф |
| **Накопление** | Линейное (1 док = 1 польза) | Экспоненциальное (через синтез) |

## 2. Проблематика
**Основные недостатки стандартного подхода:**

| Проблема | Описание | Последствие |
| :--- | :--- | :--- |
| **Повторное открытие** | Извлечение одних и тех же связей заново | Лишние затраты токенов и времени |
| **Амнезия контекста** | Нет связи между сессиями и документами | Противоречия и ошибки в выводах |
| **Фрагментарность** | Знания "застревают" в истории чатов | Информация не структурирована |

## 3. Архитектура
### 3.1. Трехуровневая модель

В основе системы лежит разделение данных по степени их «переваренности» и ответственности за их изменение.

| Уровень | Ответственность | Состав | Режим доступа |
| :--- | :--- | :--- | :--- |
| **Слой 1: Raw** | Источник истины | PDF, Статьи, Логи, Медиа | Только чтение (Immutable) |
| **Слой 2: Wiki** | Синтезированное знание | Сущности, Концепты, Индексы | Чтение/Запись (LLM-Owned) |
| **Слой 3: Schema** | Конфигурация системы | Инструкции (CLAUDE.md), Шаблоны | Общий (Shared/Config) |


## 5. Методология
### 5.2 Анатомия идеальной схемы

| Блок схемы | Содержание | Цель |
| :--- | :--- | :--- |
| **Principles** | Глобальные правила и "этика" агента | Идентичность и безопасность |
| **Structure** | Описание иерархии папок (`/raw`, `/wiki`) | Навигация и порядок |
| **Workflows** | Пошаговые SOP (Ingest, Query, Lint) | Дисциплина выполнения задач |
| **Style Guide** | Naming, YAML-frontmatter, [[Links]] | Унификация и читаемость |
