# 🏷️ Классификация связей (Link Taxonomy)

Для систематизации графа и возможности автоматизированного анализа (через Dataview) мы используем типизированные ссылки в формате `Ключ:: [[Ссылка]]`.

## 1. Иерархия C4 (Вертикальные связи)
- **contains-container::** Используется в System Context для указания на контейнеры.
- **contains-component::** Используется в Container для указания на компоненты.
- **contains-code::** Используется в Component для указания на файлы логики.
- **is-part-of::** Обратная ссылка на уровень выше.

## 2. Структура UI (Композиция)
- **parent-component::** Ссылка на компонент-обертку.
- **sub-component::** Ссылка на вложенный компонент.

## 3. Взаимодействие (Функциональные связи)
- **calls::** Прямой вызов API или функций.
- **uses-logic::** Связь компонента с его реализацией на уровне Code.
- **triggers::** Отправка событий или сигналов управления.

## 4. Ассоциации
- **relates-to::** Горизонтальная связь между равноправными модулями.
- **navigates-to::** Ссылка на оглавление или карту.
